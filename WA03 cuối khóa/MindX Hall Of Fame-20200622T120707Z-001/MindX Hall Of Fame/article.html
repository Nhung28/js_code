What should front-end programmers who are newbies put in their pockets?

Frondend dev is the one who will handle the interaction with the user directly, so I understand roughly the dev's work and skills will be equivalent and need to increase according to what the user can receive.

For example:

•In the account login interface, users need 2 input boxes and a submit button, then the frontend dev will only need basic HTML skills.

•However, when that interface is placed in the total interface of the site, with headers, navigation, footers, ... then it is natural to know the CSS.

•And when you have finished logging in, the browser does not need to reload, but the user is able to login successfully and access the content, then processing your asynchronous Javascript is indispensable. .

So here are some things I find to be better if you start to notice them.

1. What should Frontend dev's bags put in soon?

Comment when writing HTML

<!-- BEGIN div-header --> <div id="header"> <p>Viblo</p> </div> <!-- END div-header --> <!-- BEGIN div-body --> <div id="body"> ... </div> <!-- END div-body --> <!-- BEGIN div-footer --> <div id="footer"> <p>Footer</p> </div> <!-- END div-footer -->

Not much is said about the effect of commenting in code. When writing long code, it is normal to close large tags, but with comments, everything becomes much easier, right.

Bootstrap or not Bootstrap?

The most popular responsive framework. The convenience of using it has been proven in every corner of the world, as long as that dev is sitting. However, have you ever encountered the situation if the project does not use Bootstrap anymore and do not know how to write CSS yet? Mastering Bootstrap is great, but try to master your CSS skills from the start. Don't just point at Bootstrap to see how they named the class, but let's F12 up and see how they did it.

<div class="alert alert-primary" role="alert"> This is a primary alert—check it out! </div>

It's simple, 3 lines but not really easy, so if you want to customize it, master it well.

.alert { position: relative; padding: .75rem 1.25rem; margin-bottom: 1rem; border: 1px solid transparent; border-radius: .25rem; } .alert-primary { color: #004085; background-color: #cce5ff; border-color: #b8daff; }

One more thing, Bootstrap is quite heavy, and when done, surprise ... your page is no different from other people's pages. So please use Bootstrap properly, do not leave it as shown above, then when you surprise episode 2, you quit.

Semantic Markup

When Bootstrap is no longer in your pocket, getting acquainted with the Semantic Markup will help the Frontend road be less difficult.

<!-- HTML --> <!-- Bad Semantics --> <div class="article"> <div class="article_title">Title</div> <div class="the_content">Content <div class="darkbold">text</div> Content </div> </div> <!-- Good Semantics --> <article> <h1>Title</h1> <p>Content <b>Text</b> Content </p> </article>

Try to find out what BEM is and try to change the naming of your class according to the rules to see if it is worth taking with you:

<!-- HTML --> <a class="btn btn--big btn--orange"> <span class="btn__price"> $9.99</span> <span class="btn__text"> Subscribe</span> </a> <!-- CSS --> /* Một Block (khối) độc lập */ .btn {} /* Element (phần tử) con, phụ thuộc vào Block ở trên */ .btn__price {} /* Modifier (bộ điều chỉnh) thay đổi trạng thái của Block */ .btn--orange {} .btn--big {}

Optimize writing HTML and CSS code by using proper HTML tags and naming the CSS class which means it is a good habit and not difficult to get used to and learn it so try to use it early.

Javascript HTML DOM

Learning Javascript is a must for a Frontend dev, to master it and develop when used with Frameworks is really difficult. However, try to go from the most basic, do not rush too JQuery or other Frameworks. And it is important to understand the DOM carefully, everything will go far back to the DOM.

•DOM document: is responsible for storing all components in the document of the website
•DOM element: is responsible for accessing certain HTML tags through properties such as class name, id, and name of HTML tag
•DOM HTML: is responsible for changing the content value and attribute value of HTML tags
•DOM CSS: is responsible for changing the CSS format of the HTML tags
•DOM Event: is responsible for assigning events like onclick (), onload () to HTML tags
•DOM Listener: is responsible for listening for events acting on that HTML tag
•DOM Navigation: used to manage and manipulate HTML tags, showing the parent-child relationship of HTML tags
•DOM Node, Nodelist: is tasked with manipulating HTML through objects
Find out the differences between browsers

Hãy thử code của mình trên các trình duyệt khác nhau, vì không phải người dùng nào cũng có Chrome hay Firefox để vừa làm vừa test như dev đúng không. Một vài thuộc tính CSS như transition cần vendor prefixes để làm việc trên nhiều trình duyệt.

Extensions

Some extensions that I used to use in Chrome browser from the time I learned until now, good code but faster is better, right.

Wappalyzer

Visiting a beautiful website, with smooth animations, Frontend users are curious as to what technologies the site is using to do so. Wappalyzer is this tool that allows us to know the technologies used in a website

Fonts Ninja

Fontface Ninja is an extension that allows us to know the type of font used on any web page. The tool displays the name, size, line spacing and letter spacing of the font.

Page Ruler Redux

This is a tool to bring the accuracy of the distance and the size of the component between the design and HTML.

Postman

In the process of being a Frontend dev, we often have to query the API. Postman is a convenient tool to help us request APIs.

Full Page Screen Capture

Full Page Screen Capture allows capturing the current page from the beginning to the end of the page, so it is useful to capture the entire page as HTML compared to the design to see if it is correct.
